# Hard Questions Remediation Checklist

> 目标：把用户质疑转成可执行验收项，避免“只有理念没有证据”。

## A. 可运行性（Usability）

- [ ] 新用户在 10 分钟内完成 `init -> ingest -> query -> lint`
- [ ] 提供 1 份示例输入文档（中文）
- [ ] 提供 1 份示例输入文档（英文）
- [ ] 提供每步预期输出示例（成功/失败）

验收标准：
- 不需要额外口头指导，按文档一次跑通

## B. 质量（Quality）

- [ ] 建立最小评测集（10~20 份文档）
- [ ] 定义指标：实体抽取准确率
- [ ] 定义指标：关系有效率
- [ ] 定义指标：查询引用覆盖率
- [ ] 定义指标：lint 问题率
- [ ] 固化基线评测报告

验收标准：
- 每次规则/提示词改动后可重复跑回归

## C. 可靠性（Reliability）

- [ ] 写入前校验（frontmatter、命名、链接）
- [ ] 写入后自动 lint
- [ ] 失败日志可追溯（时间、输入、错误、影响页）
- [ ] 页面级备份/快照策略
- [ ] 冲突/重复/格式错误恢复手册

验收标准：
- 至少 3 类故障可复现且可恢复

## D. Schema 与兼容（Schema & Migration）

- [ ] 增加 schema version 规范
- [ ] 定义迁移策略（向后兼容窗口）
- [ ] 定义字段废弃流程（deprecation）
- [ ] 补充迁移前后检查清单

验收标准：
- 老数据升级后查询能力不下降

## E. 团队协作（Collaboration）

- [ ] 约定页面命名规范与 ownership
- [ ] PR 检查项覆盖链接完整性与 frontmatter
- [ ] 合并前强制 `lint`
- [ ] 冲突页人工确认流程

验收标准：
- 多人并行修改一周内无高危覆盖事故

## F. 隐私安全（Privacy & Security）

- [ ] 明确敏感信息类型定义（PII/密钥/凭证）
- [ ] ingest 前脱敏流程（可选）
- [ ] 模型处理边界声明（哪些数据会被处理）
- [ ] 日志与保留策略说明

验收标准：
- 能通过一次内部安全自查

## G. 信任机制（Trust & Explainability）

- [ ] 查询结果默认提供引用来源
- [ ] 低置信关系标记为待确认
- [ ] lint 检出“核心结论无来源支撑”

验收标准：
- 用户可追溯关键结论到具体源文档
