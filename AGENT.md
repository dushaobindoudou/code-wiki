# AGENT.md

本文件定义本仓库内 AI Agent 的通用执行约束，适用于 Cursor Agent 与其他协作型代码代理。

## 1. 项目定位

- 项目名称：`code-wiki`
- 当前阶段：设计与技能构建阶段（非完整应用开发）
- 目标：构建并发布 `llm-wiki` 主技能及其子技能

## 2. 工作原则

- 先读上下文，再改文件；避免盲改。
- 变更尽量小而清晰，优先可验证性。
- 不得无故重构无关文件，不得回滚用户已有修改。
- 修改后优先做最小验证（结构、格式、链接、命令可执行性）。

## 3. 命令与依赖

- Node 生态统一使用 `pnpm`。
- 不使用 `npm` / `yarn` 安装依赖或执行脚本（除非用户明确要求）。
- 若新增依赖，优先使用最新稳定版并在变更说明中写明原因。

## 4. 文档与输出规范

- 与用户沟通默认使用简体中文。
- 文档路径、命令、标识符使用反引号包裹。
- 新增文档优先遵循现有仓库结构（`docs/`、`skills/`、`tools/`）。
- 保持 README / 中文 README / Skill 文档内容一致性，不制造互相冲突描述。

## 5. 技能开发约束

- 在处理某个 Skill 前，先阅读对应 `SKILL.md`。
- Skill 文档优先描述：目标、触发条件、输入输出、执行步骤、边界条件。
- 若改动涉及多个 Skill，需明确主 Skill 与子 Skill 的职责边界。

## 6. Cursor 规则协同

- 项目级规则放在 `.cursor/rules/`。
- 规则应简短、可执行、可验证，避免空泛原则。
- 本文件与 `.cursor/rules/*.mdc` 互补：本文件偏“全局协作契约”，rules 偏“会话执行约束”。
